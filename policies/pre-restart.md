# Pre-Restart Validation - Policy

## Принцип

Нулевая толерантность к "перезапустился и не поднялся". Лучше не обновиться, чем упасть.
Перед КАЖДЫМ `run.sh restart` - обязательная проверка. Глубина зависит от масштаба изменений.

## Уровни проверки

### Уровень 1 - лёгкие изменения

Когда: новая функция, мелкий фикс, изменение текста/конфига, добавление хендлера.

Чеклист:
1. `python -m py_compile bot.py` - синтаксис OK
2. `python -c "from bot import ..."` - импорты не сломаны
3. Записать restart_reason.txt
   Формат не "короткая причина", а короткий post-restart отчет:
   - первая строка: что сделал
   - дальше 1-3 коротких bullets
   - по возможности упомянуть ключевую проверку
4. Перезапуск

### Уровень 2 - серьёзные изменения

Когда: переработка стриминга, изменение main(), модификация run_claude_streaming(),
изменение message handlers, работа с asyncio, изменение парсинга/рендеринга.

Чеклист:
1. `python -m py_compile bot.py` - синтаксис OK
2. `python -c "from bot import ..."` - импорты не сломаны
3. Тест критических функций (markdown_to_telegram_html, format_tool_notification и др.)
4. Бэкап: `cp bot.py bot.py.bak`
5. Dry-run: запустить бота, дождаться "Starting lil_worker bot", убить через 3с
6. Записать restart_reason.txt
   Формат:
   - `Что сделал:`
   - `- ...`
   - `- ...`
   - `- проверил ...`
7. Перезапуск

### Как определить уровень

Уровень 2 если затронуто ЛЮБОЕ из:
- `async def main()` или `start_polling`
- `run_claude_streaming()` или `_next_line()` или `_heartbeat()`
- Обработчики сообщений (message handlers, router)
- asyncio логика (Event, Task, wait_for)
- Импорты в начале файла
- `markdown_to_telegram_html()` или `TelegramHTMLRenderer`

Всё остальное - уровень 1.

## Скрипт валидации

Путь: `~/lil_worker/bot/validate.sh`

Запуск:
- `./validate.sh` - уровень 1 (быстрая проверка)
- `./validate.sh --deep` - уровень 2 (полная проверка с dry-run)

## При провале проверки

1. НЕ перезапускать
2. Если есть бэкап - откатить: `cp bot.py.bak bot.py`
3. Сообщить пользователю что именно сломалось
4. Предложить исправление или откат

## Rollback (если бот не поднялся после рестарта)

```bash
cp ~/lil_worker/bot/bot.py.bak ~/lil_worker/bot/bot.py
~/lil_worker/bot/run.sh restart
```

Бэкап создаётся автоматически при уровне 2. При уровне 1 бэкап опционален
(изменения мелкие, легко откатить через git).
